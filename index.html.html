1 Causa: Autenticación Anónima Deshabilitada

Firebase bloquea los accesos automáticos para evitar intrusos si no tiene encendido el protocolo de usuarios anónimos.

Cómo habilitarlo en un minuto:

  1. Abre tu Consola de Firebase y entra a tu proyecto testigos-6ecc7.
  2. En la barra izquierda haz clic en Authentication.
  3. Haz clic en la pestaña Sign-in method (Método de inicio de sesión).
  4. Haz clic en Añadir nuevo proveedor, selecciona la opción Anónimo (Anonymous), actívala y dale a Guardar.

2 Causa: Reglas de Firestore Bloqueadas (Permission Denied)

El error indica que tus reglas de base de datos impiden la lectura o escritura del canal de testigos.

Cómo actualizar tus reglas:

  1. Abre tu Consola de Firebase.
  2. Haz clic en Firestore Database en el menú izquierdo.
  3. Entra a la pestaña de Reglas (Rules).
  4. Reemplaza el código de reglas por el siguiente y dale a Publicar:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}

3 Causa: Base de Datos de Firestore no ha sido Creada

Has configurado Firebase pero aún no has inicializado el módulo de base de datos de Firestore en tu proyecto.

Cómo crear la base de datos:

  1. En la consola de Firebase del proyecto, haz clic en Firestore Database.
  2. Haz clic en el botón central de Crear Base de Datos.
  3. Selecciona la ubicación de tu servidor (ej. us-central1) y elige Comenzar en modo de prueba para evitar bloqueos iniciales.
  4. Completa el asistente.

Otras verificaciones generales:

  • Verifica que no tengas bloqueadores de publicidad (AdBlockers) o VPN activados que bloqueen peticiones a servidores de Google.
  • Asegúrate de estar conectado a una red estable de internet.
